    ліјлҘЁ мӢӨмҠө вҖ” PersistentVolume / PVC¶

    мӢӨмҠө лӘ©н‘ң¶

    • PersistentVolume(PV)кіј PersistentVolumeClaim(PVC)мқ„ мғқм„ұн•ҳкі  л°”мқёл”©н•ңлӢӨ.
    • Podм—җм„ң PVCлҘј ліјлҘЁмңјлЎң л§ҲмҡҙнҠён•ҳм—¬ лҚ°мқҙн„°лҘј мҳҒмҶҚм ҒмңјлЎң м ҖмһҘн•ңлӢӨ.
    • PodлҘј мӮӯм ңн•ҳкі  мһ¬мғқм„ұн•ҙлҸ„ лҚ°мқҙн„°к°Җ мң м§ҖлҗҳлҠ” кІғмқ„ нҷ•мқён•ңлӢӨ.

    1) StorageClass нҷ•мқё¶

    нҒҙлҹ¬мҠӨн„°м—җм„ң мӮ¬мҡ© к°ҖлҠҘн•ң StorageClassлҘј нҷ•мқён•©лӢҲлӢӨ.

    kubectl get storageclass

    нҷҳкІҪм—җ л”°лқј м¶ңл Ҙмқҙ лӢӨлҰ…лӢҲлӢӨ.

    NAME                    PROVISIONER          RECLAIMPOLICY   VOLUMEBINDINGMODE      ALLOWVOLUMEEXPANSION   AGE

azurefile-csi           file.csi.azure.com   Delete          Immediate              true                   Xd
azurefile-csi-premium   file.csi.azure.com   Delete          Immediate              true                   Xd

    NAME                 PROVISIONER          RECLAIMPOLICY   VOLUMEBINDINGMODE   AGE

    NAME                   PROVISIONER             RECLAIMPOLICY   VOLUMEBINDINGMODE      ALLOWVOLUMEEXPANSION   AGE

    кё°ліё StorageClass((default) н‘ңмӢң)к°Җ мһҲмңјл©ҙ лҸҷм Ғ н”„лЎңл№„м ҖлӢқмқ„ мӮ¬мҡ©н•  мҲҳ мһҲмҠөлӢҲлӢӨ.

    AKS StorageClass мў…лҘҳ

    StorageClass л°ұм—”л“ң нҠ№м§•
    managed-csi Azure Disk (Standard SSD) лӢЁмқј л…ёл“ң RWO, DBм—җ м Ғн•©
    managed-csi-premium Azure Disk (Premium SSD) кі м„ұлҠҘ, мҡҙмҳҒ DBмҡ©
    azurefile-csi Azure Files (SMB) м—¬лҹ¬ л…ёл“ң RWX м§Җмӣҗ, кіөмң  нҢҢмқјмҡ©

    2) л°©лІ• A вҖ” м •м Ғ н”„лЎңл№„м ҖлӢқ (PV + PVC мҲҳлҸҷ мғқм„ұ)¶

    PV мғқм„ұ¶

    pv-local.yaml нҢҢмқјмқ„ л§Ңл“ӯлӢҲлӢӨ.

    apiVersion: v1
kind: PersistentVolume
metadata:
  name: local-pv
spec:
  capacity:
    storage: 1Gi
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  storageClassName: manual
  hostPath:
    path: /tmp/k8s-lab-data    # л…ёл“ңмқҳ мӢӨм ң кІҪлЎң (Rancher DesktopмқҖ Lima VM лӮҙл¶Җ кІҪлЎң)

    kubectl apply -f pv-local.yaml
kubectl get pv

    Rancher Desktop мӮ¬мҡ©мһҗ: /tmp/k8s-lab-dataлҠ” Mac лЎңм»¬мқҙ м•„лӢҢ Lima VM лӮҙл¶Җ кІҪлЎңмһ…лӢҲлӢӨ. Finderм—җм„ңлҠ” нҷ•мқён•  мҲҳ м—ҶмҠөлӢҲлӢӨ.

    м¶ңл Ҙ:

    NAME       CAPACITY   ACCESS MODES   RECLAIM POLICY   STATUS      CLAIM   STORAGECLASS
local-pv   1Gi        RWO            Retain           Available           manual

    Kubernetes 1.29 мқҙмғҒм—җм„ңлҠ” VOLUMEATTRIBUTESCLASS, REASON, AGE м»¬лҹјмқҙ м¶”к°ҖлЎң н‘ңмӢңлҗ  мҲҳ мһҲмҠөлӢҲлӢӨ. мЈјмҡ” к°’(STATUS: Available, STORAGECLASS: manual)л§Ң нҷ•мқён•ҳл©ҙ лҗ©лӢҲлӢӨ.

    PVC мғқм„ұ¶

    pvc-local.yaml нҢҢмқјмқ„ л§Ңл“ӯлӢҲлӢӨ.

    apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: local-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 500Mi
  storageClassName: manual

    kubectl apply -f pvc-local.yaml
kubectl get pvc
kubectl get pv    # STATUSк°Җ BoundлЎң ліҖкІҪлҗЁ

    м¶ңл Ҙ:

    NAME        STATUS   VOLUME     CAPACITY   ACCESS MODES   STORAGECLASS
local-pvc   Bound    local-pv   1Gi        RWO            manual

    Kubernetes 1.29 мқҙмғҒм—җм„ңлҠ” VOLUMEATTRIBUTESCLASS, AGE м»¬лҹјмқҙ м¶”к°ҖлЎң н‘ңмӢңлҗ  мҲҳ мһҲмҠөлӢҲлӢӨ. STATUS: BoundмҷҖ VOLUME: local-pv к°’л§Ң нҷ•мқён•ҳл©ҙ лҗ©лӢҲлӢӨ.

    3) л°©лІ• B вҖ” лҸҷм Ғ н”„лЎңл№„м ҖлӢқ (Azure Disk мһҗлҸҷ мғқм„ұ)¶

    AKSм—җм„ңлҠ” PVCл§Ң м„ м–ён•ҳл©ҙ Azure Diskк°Җ мһҗлҸҷмңјлЎң мғқм„ұлҗҳкі  PVлЎң м—°кІ°лҗ©лӢҲлӢӨ. PVлҘј мҲҳлҸҷмңјлЎң л§Ңл“Ө н•„мҡ”к°Җ м—ҶмҠөлӢҲлӢӨ.

    pvc-dynamic.yaml нҢҢмқјмқ„ л§Ңл“ӯлӢҲлӢӨ.

    apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: dynamic-pvc
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: managed-csi    # Azure Disk (Standard SSD)
  resources:
    requests:
      storage: 5Gi

    kubectl apply -f pvc-dynamic.yaml
kubectl get pvc dynamic-pvc
kubectl get pv    # PV(Azure Disk)к°Җ мһҗлҸҷ мғқм„ұлҗЁ

    PVCк°Җ л°”мқёл”©лҗҳл©ҙ м•„лһҳмҷҖ к°ҷмқҙ м¶ңл Ҙлҗ©лӢҲлӢӨ:

    NAME          STATUS   VOLUME                                     CAPACITY   ACCESS MODES   STORAGECLASS   AGE
dynamic-pvc   Bound    pvc-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx   5Gi        RWO            managed-csi    30s

    AKS вҖ” PVCк°Җ Pending мғҒнғңлЎң лЁёл¬ҙлҠ” кІҪмҡ°

    managed-csiлҠ” WaitForFirstConsumer лӘЁл“ңмһ…лӢҲлӢӨ. PVCл§Ң мғқм„ұн•ң мӢңм җм—җм„ңлҠ” Azure Diskк°Җ л§Ңл“Өм–ҙм§Җм§Җ м•Ҡмңјл©°, PVC мғҒнғңк°Җ PendingмңјлЎң мң м§ҖлҗҳлҠ” кІғмқҙ м •мғҒмһ…лӢҲлӢӨ. лӢӨмқҢ лӢЁкі„(4)м—җм„ң PodлҘј л°°нҸ¬н•ҳл©ҙ Podк°Җ мҠӨмјҖмӨ„лҗң л…ёл“ң мң„м№ҳм—җ л§һм¶° Azure Diskк°Җ мғқм„ұлҗҳкі  PVCк°Җ BoundлЎң л°”лҖқлӢҲлӢӨ.

    Azure Portalм—җм„ң нҷ•мқё

    PVCк°Җ Boundлҗҳл©ҙ Azure Portal вҶ’ лҰ¬мҶҢмҠӨ к·ёлЈ№ вҶ’ MC_ лЎң мӢңмһ‘н•ҳлҠ” л…ёл“ң лҰ¬мҶҢмҠӨ к·ёлЈ№ м•Ҳм—җ kubernetes-dynamic-pvc-xxx нҳ•нғңмқҳ Managed Diskк°Җ мһҗлҸҷ мғқм„ұлҗң кІғмқ„ нҷ•мқён•  мҲҳ мһҲмҠөлӢҲлӢӨ.

    Azure Files PVC (RWX вҖ” м—¬лҹ¬ Podк°Җ лҸҷмӢңм—җ л§ҲмҡҙнҠё)¶

    Azure DiskлҠ” н•ҳлӮҳмқҳ л…ёл“ңм—җм„ңл§Ң л§ҲмҡҙнҠён•  мҲҳ мһҲмҠөлӢҲлӢӨ(RWO). м—¬лҹ¬ Podк°Җ лҸҷмӢңм—җ к°ҷмқҖ ліјлҘЁмқ„ кіөмң н•ҙм•ј н•ңлӢӨл©ҙ Azure FilesлҘј мӮ¬мҡ©н•©лӢҲлӢӨ.

    pvc-files.yaml нҢҢмқјмқ„ л§Ңл“ӯлӢҲлӢӨ.

    apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: files-pvc
spec:
  accessModes:
    - ReadWriteMany          # м—¬лҹ¬ л…ёл“ң/Podм—җм„ң лҸҷмӢң л§ҲмҡҙнҠё к°ҖлҠҘ
  storageClassName: azurefile-csi  # Azure Files (SMB)
  resources:
    requests:
      storage: 5Gi

    kubectl apply -f pvc-files.yaml
kubectl get pvc files-pvc

    Azure FilesлҠ” Immediate лӘЁл“ңмқҙлҜҖлЎң Pod м—ҶмқҙлҸ„ PVCк°Җ мҰүмӢң BoundлЎң л°”лҖқлӢҲлӢӨ:

    NAME        STATUS   VOLUME                                     CAPACITY   ACCESS MODES   STORAGECLASS    AGE
files-pvc   Bound    pvc-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx   5Gi        RWX            azurefile-csi   10s

    Azure Disk vs Azure Files м„ нғқ кё°мӨҖ

    н•ӯлӘ© Azure Disk (managed-csi) Azure Files (azurefile-csi)
    м ‘к·ј лӘЁл“ң RWO (лӢЁмқј л…ёл“ң) RWX (м—¬лҹ¬ л…ёл“ң лҸҷмӢң)
    л°”мқёл”© мӢңм җ WaitForFirstConsumer (Pod л°°нҸ¬ нӣ„) Immediate (PVC мғқм„ұ мҰүмӢң)
    м„ұлҠҘ кі м„ұлҠҘ (лЎңм»¬ л””мҠӨнҒ¬ мҲҳмӨҖ) мғҒлҢҖм ҒмңјлЎң лӮ®мқҢ (л„ӨнҠёмӣҢнҒ¬ нҢҢмқјмӢңмҠӨн…ң)
    м Ғн•©н•ң мӣҢнҒ¬лЎңл“ң DB, лӢЁмқј мқёмҠӨн„ҙмҠӨ м•ұ лЎңк·ё м§‘кі„, кіөмң  м„Өм • нҢҢмқј, л©ҖнӢ° л Ҳн”ҢлҰ¬м№ҙ м•ұ

    Azure Portalм—җм„ң нҷ•мқё

    PVCк°Җ Boundлҗҳл©ҙ Azure Portal вҶ’ мҠӨнҶ лҰ¬м§Җ кі„м • м•Ҳм—җ нҢҢмқј кіөмң (File Share) к°Җ мһҗлҸҷ мғқм„ұлҗ©лӢҲлӢӨ. мҠӨнҶ лҰ¬м§Җ кі„м •мқҖ MC_ лҰ¬мҶҢмҠӨ к·ёлЈ№ м•Ҳм—җм„ң нҷ•мқён•  мҲҳ мһҲмҠөлӢҲлӢӨ.

    4) PVCлҘј мӮ¬мҡ©н•ҳлҠ” Pod л°°нҸ¬¶

    pod-with-pvc.yaml нҢҢмқјмқ„ л§Ңл“ӯлӢҲлӢӨ.

    apiVersion: v1
kind: Pod
metadata:
  name: data-pod
spec:
  containers:
    - name: app
      image: busybox:1.36-musl
      command: ["sh", "-c", "echo 'Hello K8s Storage!' > /data/hello.txt && cat /data/hello.txt && sleep 3600"]
      volumeMounts:
        - name: data-storage
          mountPath: /data
  volumes:
    - name: data-storage
      persistentVolumeClaim:
        claimName: local-pvc    # лҳҗлҠ” dynamic-pvc
  restartPolicy: Never

    kubectl apply -f pod-with-pvc.yaml
kubectl logs data-pod

    Hello K8s Storage!

    мқҙ лӢЁкі„м—җм„ң dynamic-pvcлҠ” м•„м§Ғ Pending

    мқҙ PodлҠ” local-pvcлҘј мӮ¬мҡ©н•©лӢҲлӢӨ. dynamic-pvcлҠ” н•ҙлӢ№ PVCлҘј мӮ¬мҡ©н•ҳлҠ” Podк°Җ мӢӨм ңлЎң л°°нҸ¬лҗҳм–ҙм•ј Azure Diskк°Җ мғқм„ұлҗҳкі  PVCк°Җ BoundлЎң л°”лҖқлӢҲлӢӨ.

    dynamic-pvcк°Җ Pending мғҒнғңлЎң мң м§ҖлҗҳлҠ” кІғмқҖ м •мғҒмһ…лӢҲлӢӨ. лӢӨмқҢ 6лӢЁкі„м—җм„ң MySQL DeploymentлҘј л°°нҸ¬н•ҳл©ҙ к·ёл•Ң BoundлЎң м „нҷҳлҗ©лӢҲлӢӨ.

    kubectl get pvc dynamic-pvc   # Pending вҖ” м •мғҒ, м•„м§Ғ мӮ¬мҡ©н•ҳлҠ” Podк°Җ м—ҶмқҢ

    5) лҚ°мқҙн„° мҳҒмҶҚм„ұ нҷ•мқё¶

    PodлҘј мӮӯм ңн•ҳкі  мһ¬мғқм„ұн•ҙлҸ„ лҚ°мқҙн„°к°Җ мң м§ҖлҗҳлҠ”м§Җ нҷ•мқён•©лӢҲлӢӨ.

    Pod мӮӯм ң¶

    kubectl delete pod data-pod

    мғҲ Podм—җм„ң к°ҷмқҖ PVC мһ¬мӮ¬мҡ©¶

    pod-with-pvc-2.yaml нҢҢмқјмқ„ л§Ңл“ӯлӢҲлӢӨ.

    apiVersion: v1
kind: Pod
metadata:
  name: data-pod-2
spec:
  containers:
    - name: app
      image: busybox:1.36-musl
      command: ["sh", "-c", "cat /data/hello.txt && sleep 3600"]
      volumeMounts:
        - name: data-storage
          mountPath: /data
  volumes:
    - name: data-storage
      persistentVolumeClaim:
        claimName: local-pvc
  restartPolicy: Never

    kubectl apply -f pod-with-pvc-2.yaml
kubectl logs data-pod-2

    Hello K8s Storage!

    нҸ¬мқёнҠё: Podк°Җ мӮӯм ңлҗҳкі  мғҲ Podк°Җ мғқм„ұлҗҳм–ҙлҸ„ /data/hello.txt нҢҢмқјмқҙ к·ёлҢҖлЎң лӮЁм•„мһҲмҠөлӢҲлӢӨ.

    6) PVCлҘј Deploymentм—җ мӮ¬мҡ©¶

    мӢӨм ңлЎңлҠ” Deploymentм—җм„ң PVCлҘј мӮ¬мҡ©н•©лӢҲлӢӨ.

    MySQLмқҖ /var/lib/mysqlмқҙ л№Ҳ л””л үнҶ лҰ¬м—¬м•ј мҙҲкё°нҷ”лҗ©лӢҲлӢӨ. м•һ лӢЁкі„м—җм„ң лҚ°мқҙн„°к°Җ кё°лЎқлҗң local-pvcлҘј мһ¬мӮ¬мҡ©н•ҳл©ҙ мӢӨнҢЁн•ҳлҜҖлЎң, 3лӢЁкі„м—җм„ң мғқм„ұн•ң dynamic-pvc(Azure Disk, л№Ҳ мғҒнғң)лҘј мӮ¬мҡ©н•©лӢҲлӢӨ.

    deploy-with-pvc.yaml нҢҢмқјмқ„ л§Ңл“ӯлӢҲлӢӨ.

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: mysql-with-storage
spec:
  replicas: 1
  selector:
    matchLabels:
      app: mysql-storage
  template:
    metadata:
      labels:
        app: mysql-storage
    spec:
      containers:
        - name: mysql
          image: mysql:8.0
          env:
            - name: MYSQL_ROOT_PASSWORD
              value: testpassword
          volumeMounts:
            - name: mysql-data
              mountPath: /var/lib/mysql
      volumes:
        - name: mysql-data
          persistentVolumeClaim:
            claimName: dynamic-pvc   # 3лӢЁкі„м—җм„ң мғқм„ұн•ң Azure Disk PVC

    kubectl apply -f deploy-with-pvc.yaml
kubectl get pods -l app=mysql-storage
kubectl get pvc dynamic-pvc   # мқҙ мӢңм җм—җм„ң Pending вҶ’ BoundлЎң м „нҷҳлҗЁ

    Podк°Җ м •мғҒ кё°лҸҷлҗҳл©ҙ Azure Diskк°Җ н•ҙлӢ№ л…ёл“ңм—җ л§ҲмҡҙнҠёлҗҳкі  MySQL лҚ°мқҙн„°к°Җ мҳҒмҶҚм ҒмңјлЎң м ҖмһҘлҗ©лӢҲлӢӨ. Podк°Җ мһ¬мӢңмһ‘лҗҳкұ°лӮҳ лӢӨлҘё л…ёл“ңлЎң мқҙлҸҷн•ҙлҸ„ лҸҷмқјн•ң Azure Diskк°Җ мһҗлҸҷмңјлЎң мһ¬м—°кІ°лҗ©лӢҲлӢӨ.

    м ‘к·ј лӘЁл“ң л№„көҗ¶

    лӘЁл“ң м„ӨлӘ… мҡ©лҸ„
    ReadWriteOnce (RWO) н•ҳлӮҳмқҳ л…ёл“ңм—җм„ңл§Ң мқҪкё°/м“°кё° лҚ°мқҙн„°лІ мқҙмҠӨ
    ReadOnlyMany (ROX) м—¬лҹ¬ л…ёл“ңм—җм„ң мқҪкё° м „мҡ© м •м Ғ нҢҢмқј кіөмң 
    ReadWriteMany (RWX) м—¬лҹ¬ л…ёл“ңм—җм„ң мқҪкё°/м“°кё° NFS, кіөмң  мҠӨнҶ лҰ¬м§Җ

    лЎңм»¬(hostPath)мқҖ RWOл§Ң м§Җмӣҗн•©лӢҲлӢӨ. RWXлҠ” NFS лҳҗлҠ” нҒҙлқјмҡ°л“ң нҢҢмқј мҠӨнҶ лҰ¬м§Җ(Azure Files, EFS)к°Җ н•„мҡ”н•©лӢҲлӢӨ.

    м •лҰ¬ (лҰ¬мҶҢмҠӨ мӮӯм ң)¶

    kubectl delete pod data-pod-2
kubectl delete -f deploy-with-pvc.yaml
kubectl delete pvc local-pvc dynamic-pvc
kubectl delete pv local-pv

    PV Reclaim Policyк°Җ Retainмқҙл©ҙ PVC мӮӯм ң нӣ„м—җлҸ„ PVк°Җ Released мғҒнғңлЎң лӮЁмҠөлӢҲлӢӨ. мҲҳлҸҷмңјлЎң мӮӯм ңн•ҙм•ј н•©лӢҲлӢӨ.

    нҠёлҹ¬лё”мҠҲнҢ…¶

    мҰқмғҒ нҷ•мқё мӮ¬н•ӯ
    PVC Pending мғҒнғң StorageClass мқҙлҰ„ мқјм№ҳ м—¬л¶Җ, PV мҡ©лҹүмқҙ м¶©л¶„н•ңм§Җ нҷ•мқё
    Pod Pending (PVC кҙҖл Ё) PVCк°Җ Bound мғҒнғңмқём§Җ нҷ•мқё
    лҚ°мқҙн„° мң мӢӨ PVмқҳ Reclaim Policyк°Җ Deleteмқҙл©ҙ PVC мӮӯм ң мӢң лҚ°мқҙн„° мӮӯм ңлҗЁ
    MySQL Pod Error / CrashLoopBackOff л§ҲмҡҙнҠё кІҪлЎң(/var/lib/mysql)м—җ кё°мЎҙ нҢҢмқјмқҙ мһҲмңјл©ҙ мҙҲкё°нҷ” мӢӨнҢЁ вҖ” л№Ҳ PVCлҘј мғҲлЎң мғқм„ұн•ҙм„ң мӮ¬мҡ©
    Rancher Desktopм—җм„ң PVC Pending мң м§Җ local-path StorageClassлҠ” WaitForFirstConsumer лӘЁл“ң вҖ” PodлҘј л°°нҸ¬н•ҙм•ј PVк°Җ мғқм„ұлҗЁ

    AKS мӢӨмҠө нӣ„ лҰ¬мҶҢмҠӨ мқјкҙ„ м •лҰ¬¶

    AKS нҷҳкІҪм—җм„ң мӢӨмҠөмқҙ лҒқлӮң л’Ө мғқм„ұн•ң лӘЁл“  лҰ¬мҶҢмҠӨлҘј н•ң лІҲм—җ мӮӯм ңн•©лӢҲлӢӨ. --wait=false мҳөм…ҳмқ„ мӮ¬мҡ©н•ҳл©ҙ мӮӯм ң мҷ„лЈҢлҘј кё°лӢӨлҰ¬м§Җ м•Ҡкі  л°ұк·ёлқјмҡҙл“ңлЎң м§„н–үлҗ©лӢҲлӢӨ.

    AKSм—җм„ң PVC мӮӯм ң мӢң Azure DiskлҸ„ н•Ёк»ҳ мӮӯм ңлҗЁ

    AKSмқҳ кё°ліё StorageClass(managed-csi)лҠ” Reclaim Policyк°Җ Deleteмһ…лӢҲлӢӨ. PVCлҘј мӮӯм ңн•ҳл©ҙ м—°кІ°лҗң Azure Disk(PV)лҸ„ мһҗлҸҷмңјлЎң мӮӯм ңлҗ©лӢҲлӢӨ. лі„лҸ„лЎң Azure Portalм—җм„ң л””мҠӨнҒ¬лҘј м •лҰ¬н•  н•„мҡ”к°Җ м—ҶмҠөлӢҲлӢӨ.

    PVк°Җ Released мғҒнғңлЎң лӮЁм•„мһҲлҠ” кІҪмҡ° (Retain Policy) мҲҳлҸҷ мӮӯм ңк°Җ н•„мҡ”н•©лӢҲлӢӨ:

    kubectl delete pv $(kubectl get pv --no-headers | awk '{print $1}')

    л„Өмһ„мҠӨнҺҳмқҙмҠӨ м „мІҙ мӮӯм ңлЎң н•ң лІҲм—җ м •лҰ¬

    мӢӨмҠөмқ„ лі„лҸ„ л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җм„ң м§„н–үн–ҲлӢӨл©ҙ л„Өмһ„мҠӨнҺҳмқҙмҠӨлҘј нҶөм§ёлЎң мӮӯм ңн•ҳлҠ” кІғмқҙ к°ҖмһҘ л№ лҰ…лӢҲлӢӨ:

    kubectl delete namespace <мӢӨмҠө-л„Өмһ„мҠӨнҺҳмқҙмҠӨ> --wait=false
